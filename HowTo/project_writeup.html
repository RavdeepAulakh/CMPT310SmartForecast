1. System Explanation

1.1 What Does the System Do?

Smart Forecast is a machine learning system that predicts a student's final exam score based on four input features: hours studied, average nightly sleep hours, class attendance percentage, and previous test scores. The system trains multiple regression models, compares their performance, selects the best-performing model, and deploys it as an interactive Flask web application where users can input their own study habits and receive an instant predicted exam score.

1.2 AI Methods Used

The system employs three supervised regression algorithms from scikit-learn:

  • Linear Regression — A parametric baseline model that fits a linear relationship between features and the target. Simple, fast, and interpretable.
  • Random Forest Regressor — An ensemble of decision trees that captures non-linear relationships and feature interactions. Tuned via grid search over tree count, depth, and split criteria.
  • Support Vector Regression (SVR) — A kernel-based method that finds a function within an epsilon-tube around predictions. Uses StandardScaler preprocessing and is tuned over C, epsilon, and kernel type (linear vs. RBF).

All models are evaluated using 5-fold cross-validation during hyperparameter tuning (GridSearchCV), with R² as the scoring metric. Final evaluation uses MSE, RMSE, and R² on a held-out 20% test set.

The system follows a standard ML pipeline, implemented across modular Python scripts:

(drop ID, split 80/20) → EDA

Visualization → Flask Web
Deployment

Pipeline Components

Each stage is implemented in a separate module under src/:

  • data_preprocessing.py — Loads data/exams.csv, drops student_id, separates features and target, performs 80/20 train-test split with random_state=42.
  • eda.py — Generates histograms, correlation heatmaps, and scatter plots to understand data distributions and feature-target relationships.
  • baseline_model.py — Trains a Linear Regression model and reports MSE, RMSE, and R².
  • model_experiments.py — Trains Random Forest and SVR with GridSearchCV (5-fold CV), reports best hyperparameters and test metrics.
  • final_model.py — Selects the best model by R², retrains on the full training set, evaluates on the test set, and generates diagnostic plots.

1.5 Limitations

Small Dataset (200 samples). With only 200 records and 4 features, the models have limited data to learn complex patterns. This increases the risk of overfitting, especially for Random Forest which can memorize small datasets. Mitigation: We use 5-fold cross-validation during hyperparameter tuning and hold out a 20% test set for unbiased evaluation.
Linear Regression as Deployed Model. The Flask deployment uses Linear Regression rather than the potentially higher-performing Random Forest or SVR. While Linear Regression is simpler and faster for real-time inference, it cannot capture non-linear relationships between features and exam scores. Mitigation: The pipeline trains and compares all three models so the trade-off is understood. The deployment model can be swapped by changing the model class in frontend/train_model.py.
No Feature Engineering. The system uses raw features without creating interaction terms, polynomial features, or domain-specific transformations (e.g., study efficiency = score/hours). This may limit model performance. Mitigation: Grid search over model hyperparameters partially compensates by allowing the model to find optimal complexity.
SVR Sensitivity to Scaling. SVR requires feature scaling to perform well. Without the StandardScaler pipeline, SVR would produce poor results. This was identified during development and addressed by wrapping SVR in a scikit-learn Pipeline.

3.2 Datasets

The project uses a single dataset: data/exams.csv, a synthetic exam score dataset with 200 student records and 6 columns (student_id, hours_studied, sleep_hours, attendance_percent, previous_scores, exam_score). This dataset was provided for the course project and is included in the repository.
